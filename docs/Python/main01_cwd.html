工作目录

  • 工作目录指Python解释器在执行代码时所在的目录（类似使用pwd命令所输出的内容）。

  • .py文件内的所有相对路径以及本地库的导入都基于当前工作目录。

  • 当报错提醒找不到指定文件或模块时可以考虑是不是工作目录设置有问题。

  • 使用os.getcwd()或Path.cwd()可以查看当前工作目录。

  • 默认工作目录并非文件所在目录，而一般是启动Python解释器时的位置：

    • 对于Xshell、MobaXterm等终端，这里的“位置”对应pwd的输出；

    • 对于VS Code，这里的“位置”对应连接服务器所选择的文件夹路径。

调整工作目录

  • 通常使用os.chdir调整工作目录：

    • 对于.py，有通用的os.chdir(Path(__file__).resolve().parent)调整工作目录为文件实际所在目录；

    • 对于.ipynb，由于Jupyter Notebook等交互式环境中的代码是逐块执行的，不将其视为整体文件，没有明确的__file__，因此需要显式地给出绝对路径。

  • 示例代码中（在此之前作了软链接）：

    • /data3/XiaAnRen/vscode/python_3.11是VS Code连接服务器所选择的文件夹路径；

    • /public/home/XiaAnRen/data3/vscode/python_3.11/Tutorial/tutorial/stdlibs是使用VS Code复制软链接后文件的绝对路径再删去文件名所得到的结果；

    • /public/home/XiaAnRen/share/tutorial/stdlibs是文件的实际路径。

import os
from pathlib import Path

Path.cwd()

PosixPath('/data3/XiaAnRen/vscode/python_3.11')

rootpath = (
    "/public/home/XiaAnRen/data3/vscode/python_3.11"
    "/Tutorial/tutorial/stdlibs"
)
os.chdir(Path(rootpath))

Path.cwd()

PosixPath('/public/home/XiaAnRen/share/tutorial/stdlibs')
